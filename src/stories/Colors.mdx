# Color System

Our color system is specifically designed for pool maintenance applications, emphasizing safety, clarity, and intuitive understanding. Colors communicate status, priority, and context throughout the interface.

## üé® Color Philosophy

### Safety-First Design

Colors in pool maintenance carry critical safety implications. Our palette ensures:

- **Immediate Recognition**: Critical states are instantly identifiable
- **Universal Understanding**: Colors follow established safety conventions
- **High Contrast**: All combinations meet WCAG 2.1 AA standards
- **Color-blind Friendly**: Information isn't conveyed by color alone

### Semantic Meaning

Each color has specific meaning in the pool maintenance context:

- **Blue**: Water, cleanliness, system status
- **Green**: Safe, optimal, good conditions
- **Yellow/Amber**: Caution, attention needed, maintenance
- **Red**: Danger, critical issues, immediate action required
- **Gray**: Inactive, disabled, neutral information

## ‚ôø Accessibility Standards

### Contrast Ratios

All color combinations meet WCAG 2.1 AA standards:

**Text Contrast Requirements**

- Normal text: minimum 4.5:1 contrast ratio
- Large text (18pt+): minimum 3:1 contrast ratio
- Icons and graphics: minimum 3:1 contrast ratio

**Tested Combinations**

```css
/* ‚úÖ PASS - 4.52:1 ratio */
.text-gray-700-on-white {
  color: #374151;
  background: #ffffff;
}

/* ‚úÖ PASS - 5.74:1 ratio */
.text-white-on-pool-blue {
  color: #ffffff;
  background: #0ea5e9;
}

/* ‚úÖ PASS - 4.56:1 ratio */
.text-white-on-success {
  color: #ffffff;
  background: #10b981;
}

/* ‚ùå FAIL - Would be insufficient */
.text-gray-400-on-white {
  color: #9ca3af;
  background: #ffffff;
} /* 2.8:1 */
```

### Color-blind Considerations

**Pattern Support**
Never rely solely on color to convey information:

- Use icons alongside color coding
- Include text labels for status
- Provide pattern or shape differentiation
- Test with color-blind simulation tools

**High Contrast Mode**
Colors adapt gracefully to system high contrast settings:

```css
@media (prefers-contrast: high) {
  .status-warning {
    background-color: #000000;
    color: #ffffff;
    border: 2px solid #ffffff;
  }
}
```

## üî¨ Color Science

### Chemical Level Color Mapping

Our color choices align with scientific and industry standards:

**Free Chlorine (ppm)**

- 0.0-0.4: Red (Critical - pool closure)
- 0.5-0.9: Red (Below safe minimum)
- 1.0-3.0: Green (Optimal range)
- 3.1-5.0: Yellow (Acceptable but high)
- 5.0+: Red (Dangerous levels)

**pH Scale**

- < 7.0: Red (Acidic, equipment damage risk)
- 7.0-7.1: Yellow (Low but acceptable)
- 7.2-7.6: Green (Optimal for chlorine effectiveness)
- 7.7-7.8: Yellow (High but acceptable)
- > 7.8: Red (Alkaline, ineffective sanitization)

### Temperature Ranges

- < 70¬∞F (21¬∞C): Blue (Cool)
- 70-82¬∞F (21-28¬∞C): Green (Comfortable)
- 83-86¬∞F (28-30¬∞C): Yellow (Warm)
- > 86¬∞F (30¬∞C): Red (Too hot, bacterial risk)

## üìñ Best Practices

### Do's

‚úÖ **Use semantic color names** instead of descriptive ones
‚úÖ **Test color combinations** for accessibility compliance
‚úÖ **Provide non-color alternatives** for important information
‚úÖ **Consider color-blind users** in all design decisions
‚úÖ **Use consistent colors** for the same meaning across the app
‚úÖ **Test on actual devices** and different lighting conditions

### Don'ts

‚ùå **Don't use color as the only way** to convey critical information
‚ùå **Don't use too many colors** in a single interface
‚ùå **Don't ignore contrast ratios** for decorative elements
‚ùå **Don't assume color perception** is the same for all users
‚ùå **Don't use colors that conflict** with established safety standards

### Color Combinations to Avoid

- Red and green together (color-blind accessibility)
- Low contrast combinations (accessibility)
- Too many bright colors simultaneously (visual fatigue)
- Cultural color conflicts (red = stop in most cultures)
